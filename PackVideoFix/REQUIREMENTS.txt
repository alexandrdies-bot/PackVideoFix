# Требования

## Функциональные (MVP)
- Превью видео в окне приложения
- Запуск/остановка камеры
- Запись клипа по событию (Enter/скан)
- Длительность клипа задаётся в настройках
- Сохранение клипов по папкам: дата/станция
- Лог в SQLite: barcode, filePath, startedAt, finishedAt, status, error
- Поиск по barcode (частичное совпадение)
- Открытие клипа двойным кликом

## Нефункциональные
- Простота для оператора: 1 поле + Enter
- Стабильность, отсутствие утечек памяти
- Настройки через appsettings.json (в репозитории только example)
- Не хранить пароли/RTSP в git

## Будущее
- IP камера (RTSP)
- mp4/h264 при необходимости
- 2 камеры, преролл 10–20 секунд
